Procedural Programming

Procedural programming is a programming paradigm (style of writing software) based on the idea of organizing code into procedures, also called functions, subroutines, or routines. These procedures contain a sequence of steps that are executed in order — like a recipe. Some key ideas of procedural programming include: Step-by-step execution, variables and data, control structures, focus on actions, and procedures/functions.

C (1972)

Systems language designed at Bell Labs by Dennis Ritchie; portable and close to hardware.

Influence

C inspired C++, C#, Java, Objective-C, Rust (via safety focus), and Go (via simplicity).

Pascal (1970)

Designed by Niklaus Wirth with emphasis on clarity and teaching structured programming.

Fortran & COBOL

Among the earliest high-level languages: scientific computing (Fortran) and business processing (COBOL).
